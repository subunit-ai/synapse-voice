# Sonar v0.10.0 — Full UI Redesign Plan
**Erstellt:** 2026-05-18 | **Trigger:** TJ "die UI ist essenziell so ist es jedes Mal ein pain"
**Status:** Plan v1, freigegeben für Implementation

## Problem (jetziger Stand v0.9.18)
- Mehrere separate Fenster (Settings / History / Meetings / Onboarding) — kein „one cohesive app" Gefühl
- Settings-Tabs zu klein + unübersichtlich
- Plan-Badge nur sichtbar wenn Hauptfenster offen UND Server erreichbar
- Account-Status versteckt 2-Clicks-tief (Settings → Account)
- Modale Dialoge unterbrechen den Flow
- Touch-Targets zu klein für Tablet-Use (Erik Win-ARM Surface)

## Ziel
**Single-Window Hub** — Linke Sidebar mit Sektionen, rechte Content-Pane wechselt inline. Profil-Icon top-right für Always-On Account-Sichtbarkeit. Plan-Badge im Header. Settings = inline-Seite, kein Dialog.

## Sektionen

### 1. Home (default)
- Hero: Local/Cloud big-toggle (existing widget, bigger 480×80px)
- Mode-Pills inline (AUTO/INSTANT/FAST/QUALITY) sichtbar wenn Cloud-Subunit
- 3 Stat-Karten (TRANSCRIBED / AUDIO PROCESSED / TIME SAVED)
- Letzte 5 Transkriptionen inline (kein History-Window aufmachen)
- Big CTA „🔴 Meeting starten"

### 2. Verlauf (History)
- Full transcription list mit Suche + Date-Filter
- Inline-Preview, Copy/Paste-Buttons
- Mode-Used-Badge + Tier-Badge pro Eintrag
- Bulk-Delete / Export-Selection

### 3. Meetings
- Meeting-Cards (≥240s Recordings)
- Action-Items + Decisions inline
- Recap-Email + Export-Buttons
- QR-Checkin-Status (für v0.9.0+ Meetings)

### 4. Vocabulary
- Großer Table: Term | Sounds Like | Aliases | Category | Actions
- Add-Row-Inline + Edit-In-Place
- Default-Seed-Entries (v0.9.17) markiert
- CSV-Import/Export

### 5. Settings
Sub-Tabs (horizontal oben):
- **Allgemein** — Hotkey, Mic-Device, Sprache, UI-Theme, Autostart
- **Transkription** — Mode-Default, Cleanup, DACH-Format, Quality
- **Overlay** — Use Orb, Style, Color, Size, Position, Auto-Hide
- **Sound** — Start/Stop sounds, Volume
- **Privacy/DSGVO** — Server-Region, Retention, AVV-Download, „Alle Meetings löschen"

Save-Button im Footer der Sektion. Keine Modal-Dialoge mehr.

### 6. Hilfe / About
- Version + Updates
- Doku-Links
- Diagnose-Daten kopieren (Codex P1-Feature)
- License / Credits

## Header (Top-Bar)

```
[⌬ Sonar v0.10.0]    ←   →    [● PRO]  [(E)]  [_ □ ✕]
   Brand + Version            Plan-Badge  Profile  Window
```

- **Plan-Badge:** Farb-gecodet pro Tier
  - FREE: grau
  - TRIAL: orange (mit Restzeit „5d" Suffix)
  - PRO: cyan (Brand-Color)
  - ENTERPRISE: violet
  - OPERATOR: violet+star
- **Profile-Avatar:** Rundes Icon mit Email-Initial (z.B. „E" für Erik). Click → springt zur Account-Section.
- **Window-Controls:** Native Windows / macOS / Linux Controls. Auf Linux + tray-Mode: nur Close (minimiert zur tray).

## Backward Compatibility

- **Config-Schema unverändert** — alle bestehenden Keys bleiben gültig
- **Tray-Icon-Behavior unverändert**
- **Hotkey-Pipeline unverändert**
- **Recording-Pipeline unverändert**
- Alte `SettingsDialog`-Klasse bleibt im Code (deprecated) für Emergency-Rollback per Feature-Flag `use_legacy_settings_dialog: bool = False`

## Risk + Mitigations

| Risk | Mitigation |
|------|------------|
| Win-ARM Qt Layout-Bugs | Test früh + oft auf Erik's Tablet (jede Phase) |
| Großer Diff = Review-Aufwand | Codex Cloud-Review nach Phase 2 |
| Breaking-Change für Power-User | Feature-Flag `use_legacy_main_window` als Notausgang |
| Performance-Regress (mehr Widgets) | Lazy-Load Sektionen (nur das aktive bleibt im Memory) |

## Deferred (NICHT in v0.10.0)

- Tray-Icon-Refactor (separate Story)
- Onboarding-Wizard-Redesign (kommt nach v0.10.0, eigener Pass)
- Mobile / Touch-Optimized Layout (eigene App, nicht Desktop)
- Subunit-App Integration (separate App)
